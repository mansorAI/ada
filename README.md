# مصروفاتي 💸

تطبيق ويب لتتبع المصروفات — متعدد المستخدمين مع نظام مجموعات وصلاحيات تفصيلية.

🌐 **الموقع:** [masrofati.vercel.app](https://masrofati.vercel.app)

---

## المميزات

- **حسابات معزولة** — كل مستخدم يرى بياناته الخاصة فقط عند التسجيل
- **تسجيل الدخول** بالبريد الإلكتروني عبر Supabase Auth
- **نظام المجموعات** — أنشئ مجموعة (عائلة، أصدقاء...) أو انضم برمز دعوة
- **إضافة أعضاء بالاسم أو البريد** — بحث مباشر بدون رمز دعوة
- **صلاحيات تفصيلية لكل عضو** — مالك القروب يحدد لكل شخص:
  - رؤية المصروفات / تعديلها
  - رؤية الالتزامات / تعديلها
  - دور: مشاهد أو محرر
- **تسجيل المصروفات** يدوياً أو صوتياً
- **فئات ديناميكية** تُنشأ تلقائياً عند إضافة أي مصروف
- **الالتزامات الشهرية** مع جدول أصناف تفصيلي
- **متابعة المخزون**: كامل 🟢 / النصف 🟡 / الربع 🟠 / نفذ 🔴 / كمية مخصصة
- **تحليل الفواتير بالصورة** عبر Claude AI
- **الميزانية والمتبقي** مع قائمة تفصيلية للمبالغ
- **بداية الشهر المخصصة** أو التقويم التلقائي
- **المقارنة الشهرية** للالتزامات مع مؤشرات الارتفاع والانخفاض
- **تقارير وإحصائيات** — أسبوعية وشهرية وكاملة
- **نصائح مالية** تُولَّد تلقائياً بناءً على أنماط الإنفاق
- **واجهة عربية RTL** مع خط Tajawal — تصميم داكن

## الصفحات

| الصفحة | الوصف |
|--------|-------|
| الرئيسية | الإجمالي + الميزانية + المتبقي + شريط الشهر |
| تفاصيل الفئة | كل إدخالات فئة مصروف معينة |
| الالتزامات | كروت الالتزامات + مقارنة شهرية + أرشيف |
| تفاصيل الالتزام | جدول أصناف + تقارير الفئات + المخزون |
| المجموعات | إنشاء / انضمام / إدارة الأعضاء والصلاحيات |
| التقارير | إحصائيات + تقرير شامل |
| المهام | قريباً |

## إدارة المجموعات

### إنشاء مجموعة
1. افتح صفحة **المجموعات** من القائمة
2. اكتب اسم المجموعة → اضغط **إنشاء**

### إضافة عضو
1. اضغط **👥 الأعضاء** على المجموعة
2. ابحث باسم الشخص أو بريده الإلكتروني
3. اضغط **إضافة**

### تحديد صلاحيات العضو
1. اضغط على اسم العضو في قائمة الأعضاء
2. حدد الصلاحيات بالخانات:
   - ✅ رؤية المصروفات
   - ✅ تعديل المصروفات
   - ✅ رؤية الالتزامات
   - ✅ تعديل الالتزامات
3. اختر الدور: مشاهد أو محرر
4. اضغط **حفظ الصلاحيات**

## إضافة مصروف

اضغط زر **+** في شريط التنقل:
- 🎙️ **صوتي** — قل: `"قهوة 15"` أو `"بنزين 200 ريال"`
- ✏️ **يدوي** — اكتب الاسم والمبلغ

---

## الالتزامات الشهرية

أضف فئاتك الثابتة (إيجار، كهرباء، أغراض البيت...)، ثم اضغط على أي التزام:

| الإجراء | الطريقة |
|---------|---------|
| إضافة صنف يدوي | اسم + فئة + كمية + سعر |
| إضافة صنف صوتي | قل: "شامبو 25 ريال" |
| تحليل فاتورة | التقط صورة → يستخرج الأصناف تلقائياً |
| تعديل صنف | اضغط على الصف في الجدول |
| تحديث المخزون | اضغط زر المخزون في الجدول |
| تسجيل دفعة الشهر | يحسب الإجمالي تلقائياً |

## قاعدة البيانات (Supabase)

| الجدول | الوصف |
|--------|-------|
| `profiles` | ملفات المستخدمين |
| `groups` | المجموعات + رمز الدعوة |
| `group_members` | الأعضاء + الدور + الصلاحيات التفصيلية |
| `group_settings` | إعدادات الشهر لكل مجموعة |
| `expenses` | المصروفات اليومية |
| `commitments` | الالتزامات الشهرية |
| `commitment_items` | أصناف كل التزام |
| `budget_entries` | إدخالات الميزانية |
| `audit_log` | سجل التعديلات |

### Migration مطلوب (شغّله مرة واحدة في Supabase SQL Editor)

```sql
-- صلاحيات تفصيلية للأعضاء
alter table public.group_members
  add column if not exists can_view_expenses    boolean default true,
  add column if not exists can_edit_expenses    boolean default false,
  add column if not exists can_view_commitments boolean default true,
  add column if not exists can_edit_commitments boolean default false;

-- البحث عن المستخدمين لإضافتهم للمجموعة
drop policy if exists "profiles_select" on public.profiles;
create policy "profiles_select" on public.profiles for select
  using (auth.uid() is not null);
```

## ملاحظات تقنية

- **عزل البيانات:** كل مستخدم له مفتاح localStorage خاص (`msf_v3_{userId}`) — لا تتداخل بيانات المستخدمين على نفس الجهاز
- **مفتاح Claude API:** يُحفظ محلياً فقط (لا يُشارَك مع أعضاء المجموعة)
- **البيانات حالياً:** تُخزَّن في localStorage لكل مستخدم — مشاركة البيانات بين أعضاء المجموعة عبر Supabase قيد التطوير
